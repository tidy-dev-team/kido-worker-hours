# WorkHours — תיעוד פונקציונליות מלא

> מערכת ניהול שעות כוח אדם | Vanilla JS + HTML/CSS | אחסון: `localStorage` (מפתח: `wh-state-v3`)

---

## מבנה הנתונים (State)

```js
state = {
  clients: [],           // רשימת לקוחות
  employees: [],         // רשימת עובדים
  matrix: {},            // הקצאות: { monthKey: { empId: { clientId: hours } } }
  monthSetup: {},        // הגדרות חודש: { monthKey: { workDays, ... } }
  vacations: {},         // ימי חופשה: { monthKey: { empId: days } }
  weeklySchedule: {},    // סידור שבועי: { monthKey: { empId: { day: [clientIds] } } }
  activeMonths: [],      // חודשים פעילים (YYYY-MM)
  currentMonth: ''       // חודש נבחר
}
```

## ישויות

### לקוח (Client)

| שדה | סוג | תיאור |
|-----|-----|--------|
| `id` | string | מזהה ייחודי (`c{timestamp}`) |
| `name` | string | שם הלקוח |
| `type` | `retainer` / `project` / `internal` | סוג לקוח |
| `monthlyHours` | `{ mk: hours }` | שעות מוזמנות לכל חודש |
| `hoursBank` | number | (פרויקטים) תקציב שעות כולל |
| `billedHours` | `{ mk: hours }` | (פרויקטים) שעות שחויבו לכל חודש |
| `active` | boolean | פעיל/לא פעיל |
| `weeklyDay` | `number[]` | ימי השבוע לפגישות ויקלי (0=ראשון … 4=חמישי) |

---

### עובד (Employee)

| שדה | סוג | תיאור |
|-----|-----|--------|
| `id` | string | מזהה ייחודי (`e{timestamp}`) |
| `name` | string | שם מלא |
| `role` | string | תפקיד (אופציונלי) |
| `email` | string | כתובת מייל לשליחת הקצאות |
| `slackWebhook` | string | Webhook URL לשליחת Slack |
| `scope` | number (1–100) | אחוז משרה |
| `visible` | boolean | הצג במטריצה ובסידור שבועי |
| `monthlyHours` | `{ mk: hours }` | עקיפת שעות ידנית לחודש |
| `preferredClients` | `string[]` | מזהי לקוחות מועדפים לפיזור |

## עמודים

---

### 1. מבט על (Overview)

**מטרה:** דאשבורד הנהלה — KPI, גרפים והתראות

**כרטיסי KPI:**
- קיבולת כוללת (שעות)
- שעות מוזמנות (לקוחות)
- ניצולת % (מוקצה / קיבולת)
- מספר עובדים פעילים
- מספר לקוחות פעילים (חיצוניים)
- שעות שאבדו לחופשות וחגים

**גרפים:**
- **גרף עמודות:** לקוחות — מוזמן מול מוקצה
- **גרף עוגה:** שעות לפי סוג לקוח (Retainer / Project / Internal)
- **גרף קו:** טרנד שעות מוזמנות מול קיבולת ב-6 חודשים אחרונים

**התראות אוטומטיות:**
- עובדים עם ניצולת יתר (>100%)
- עובדים עם ניצולת נמוכה (<50%)
- עובדים ללא הקצאה עם קיבולת קיימת
- לקוחות ללא כיסוי (אפס הקצאה)
- פרויקטים עם בנק שעות נמוך (<20 שעות)
- השוואה לחודש הקודם

### 2. לקוחות (Clients)

**מטרה:** ניהול לקוחות, חוזים ושעות

**טבלה:**
- שם לקוח
- סוג (עם תג צבעוני)
- מצב פעיל (checkbox)
- שעות מוזמנות החודש (עריכה ישירה)
- כפתורי עריכה ומחיקה
- הצג/הסתר לקוחות לא פעילים

**מודל הוספה / עריכת לקוח:**

| שדה | תנאי |
|-----|------|
| שם לקוח | חובה |
| סוג לקוח | retainer / project / internal |
| שעות מוזמנות לכל 12 חודשים | retainer / internal |
| כפתור "החל על כל החודשים" | retainer / internal |
| בנק שעות | project בלבד |
| שעות מחויבות לחודש | project בלבד |
| ימי ויקלי (checkboxes א-ה) | כל הסוגים חוץ מ-internal |
| שיוך עובדים (preferredClients) | כל הסוגים |

**פעולות:**
- `saveClient` — שמירה / יצירה עם ולידציה
- `toggleClientActive` — הפעלה/השבתה
- `deleteClient` — מחיקה קבועה
- `updateClientHours` — עדכון שעות ישיר בתא

**לוגיקה עסקית:**
- פרויקט: שעות מחויבות מנכות מהבנק
- שינוי סוג מסתיר/מציג שדות רלוונטיים
- לקוח internal לא מופיע בסידור שבועי

---

### 3. עובדים (Employees)

**מטרה:** ניהול כוח אדם ושליחת הקצאות

**תצוגות:**
- **טבלה:** שם, תפקיד, שעות זמינות, ימי חופשה, פעולות
- **כרטיסיות:** כרטיס לכל עובד עם פירוט לקוחות

**מודל הוספה / עריכת עובד:**
- שם, תפקיד, מייל, Slack Webhook
- סקופ (1–100%) עם כפתורי קיצור: 25 / 50 / 60 / 75 / 80 / 100
- טבלת שעות לכל חודש (מחושבות אוטומטית, ניתנות לעקיפה ידנית)
- בחירת לקוחות מועדפים (multi-select checkboxes)

**שליחת הקצאה:**
- מודל עם הודעת הקצאה ניתנת לעריכה
- כפתור **מייל** (פותח mailto עם ההודעה)
- כפתור **Slack** (POST ל-webhook)
- כפתור **העתק** לקליפבורד

**שליחה קבוצתית:**
- שלח לכל העובדים בתור (300ms בין כל שליחה)
- מוצג סטטוס לכל עובד (מייל / Slack / חסר פרטי קשר)

**פורמט הודעת הקצאה:**
```
היי [שם],
הנה הקצאת השעות שלך לחודש [חודש]:

• לקוח א: Xh
• לקוח ב: Yh
...

סה״כ: allocated/available h
```

**פעולות:**
- `toggleEmpVisibility` — הצג/הסתר במטריצה
- `toggleAllEmployees` — הצג/הסתר הכל
- `updateEmpHours` — עדכון שעות ידני
- `updateEmpVacDays` — עדכון ימי חופשה
- `resetEmpHours` — חזרה לחישוב אוטומטי
- `deleteEmployee` — מחיקה קבועה

### 4. מטריצת הקצאות (Allocation Matrix)

**מטרה:** הקצאת שעות מעובדים ללקוחות באופן ויזואלי ואינטראקטיבי

**מבנה הטבלה:**
- **שורות:** עובדים (מסוננים לפי `visible`)
- **עמודות:** לקוחות פעילים
- **תאים:** input מספרי לשעות

**ראש עמודה — לקוח:**
- שם לקוח (קצוץ אם >11 תווים)
- `מוקצה / מוזמן h` עם צבע דינמי:
  - ירוק: מדויק
  - צהוב: חסר
  - אדום: עודף
  - אפור: 0 שעות

**ראש שורה — עובד:**
- שם עובד (ניתן ללחיצה לסינון)
- `מוקצה / זמין`
- `X/6` לקוחות פעילים
- ★ אם יש לקוחות מועדפים
- ⚠ אם מוקצה > זמין

**שורת סיכום (footer):**
- סכום מוקצה לכל לקוח עם צביעה

**כרטיסי KPI מעל המטריצה:**
- שעות לקוחות (מוזמן החודש)
- קיבולת עובדים (N עובדים פעילים)
- ניצולת % (N h מוקצה)
- פער קיבולת (קיבולת עודפת / חסר קיבולת)

**פאנל פעולות:**

| קבוצה | פעולה | תיאור |
|-------|--------|--------|
| ניהול חודש | פיזור אוטומטי | מפזר שעות לפי preferredClients + קיבולת |
| ניהול חודש | נקה מטריצה | מאפס כל ההקצאות (דורש אישור) |
| תצוגה | כרטיסיות / טבלה | מחליף בין מצבי תצוגה |

**תצוגות:**
- **טבלה:** גריד סטנדרטי גלילה
- **כרטיסיות עובדים:** כרטיס לכל עובד עם ברי הקצאה ללקוחות
- **כרטיסיות לקוחות:** כרטיס לכל לקוח עם רשימת עובדים

**פיזור אוטומטי — לוגיקה:**
1. מחשב שעות זמינות לכל עובד (`getEmpHours` — מנוכות חופשות + scope)
2. מחשב מכסה נותרת לכל לקוח
3. מפזר רק בין `preferredClients` של כל עובד
4. שעות תמיד בכפולות של 5 (`_split5`)
5. לא עולה על מכסת הלקוח לעולם
6. לא מעל 6 לקוחות לעובד
7. שאריות שלא חולקו — מוצגות בהתראה לאחר הפיזור
8. **אין Phase 2** — שעות שלא מולאו נשארות ריקות

**`_split5` (אלגוריתם עיגול ל-5):**
- מחלק `total` לחלקים שהם כפולות של 5
- משתמש בשיטת largest-remainder
- מבטיח שסכום = `floor(total/5)*5`

**התנהגות תא:**
- עדכון live: בזמן הקלדה מתעדכן סכום העמודה והשורה
- אחרי שינוי: שמירה אוטומטית ב-state
- מוגבל: לא ניתן להוסיף לקוח ה-7+ (תא מוגבל)

### 5. סידור שבועי (Weekly Schedule)

**מטרה:** חלוקת עבודה יומית לפי עובדים ולקוחות

**פריסה:**
- **טאבים:** ניווט בין שבועות החודש (שבוע 1, שבוע 2, ...)
- **עמודות:** ימי עבודה של השבוע הנבחר (א-ה + תאריך)
- **שורות:** עובדים עם שעות מוקצות / זמינות

**תא (Cell):**
- מציג שמות לקוחות כ"פילים" (pills)
- ★ על לקוח שיש לו ויקלי ביום הזה (`weeklyDay`)
- מסגרת כתומה אם נערך ידנית (ורק אם יש בחירה ידנית)
- לחיצה פותחת popover

**Popover בחירת לקוחות:**
- מציג את כל `preferredClients` של אותו עובד
- checkboxes — בחירה / ביטול בחירה מיידי
- כותרת: "לקוחות של [שם עובד]"
- סגירה בלחיצה מחוץ ל-popover

**כפתורים:**
- **נקה סידור** — מאפס את כל הסידור לחודש (דורש אישור), כל התאים ריקים
- **פיזור אוטומטי** — מחלק לקוחות לימים לפי הקצאות מטריצה + ויקלי

**לוגיקה אוטומטית (`buildAutoMapMulti`):**
1. **Pin ויקלי:** לקוח עם `weeklyDay` — כל ימי החודש שתואמים ליום הזה מוקצים לו
2. **פיזור יחסי:** ימים שנותרו מחולקים לפי יחס שעות במטריצה (largest-remainder)
3. **מיזוג:** עקיפות ידניות (`weeklySchedule`) גוברות על האוטומטי

**שמירה:**
- `state.weeklySchedule[mk][eid][day] = [clientId, ...]`
- עקיפה ידנית: `normWS(val)` → מחזיר תמיד מערך
- `wsToggleClient` — toggle לקוח + אתחול אוטומטי אם אין שמירה

### 6. הגדרות (Settings)

**מטרה:** ניהול חודשים וייצוא נתונים

**טבלת חודשים:**
- checkbox לבחירה לייצוא
- שם חודש + תג "נוכחי"
- מספר לקוחות פעילים
- מספר עובדים פעילים
- שעות מוזמנות / מוקצות / ניצולת
- כפתור **XLS** (ייצוא חודש בודד)
- כפתור **מחק** (מושבת לחודש האחרון)

**ייצוא מרובה:**
- בחר הכל / בטל הכל
- כפתור "ייצא לאקסל" — מייצא את החודשים הנבחרים

**Excel — מבנה הגיליון (4 שיטים):**
1. **סיכום:** חודש, לקוחות, עובדים, שעות, ניצולת
2. **פירוט עובדים:** קיבולת, הקצאה, ניצולת, חופשות, לקוחות פעילים
3. **פירוט לקוחות:** סוג, שעות, מוקצה, ניצולת, פיזור עובדים
4. **מטריצה:** גריד מלא עובד × לקוח

### 7. ניהול חודש (Month Setup Modal)

**פתיחה:** כפתור "חודש חדש" בסרגל ניווט

**גריד לוח שנה:**
- כותרות ימים (א-ש, RTL)
- תאים צבועים:
  - כחול: יום עבודה (א-ה)
  - אדום: סופ"ש (שישי-שבת)
  - צהוב: חג עברי
  - אפור: מחוץ לחודש
- לחיצה על תא — toggle יום עבודה / לא

**ימי חגים מחושבים אוטומטית:**
ראש השנה, יום כיפור, סוכות, שמחת תורה, חנוכה, פורים, פסח, שבועות, ל"ג בעומר, ט"ו בשבט

**שעות חופשה לעובדים:**
- רשימת עובדים עם שדה ימים
- חישוב השפעה: ימים × 7 שעות

**הוספת לקוחות חדשים (inline):**
- שם, סוג, שעות לחודש, שיוך עובדים
- שורות מרובות (כפתור "הוסף עוד")

## מנגנוני חישוב

### שעות עובד לחודש

```
calcAutoHours(mk)       = workDays(mk) × 7
calcAutoHoursForEmp     = calcAutoHours × (scope/100) - (vacDays × 7)
getEmpHours(emp, mk)    = monthlyHours[mk] || calcAutoHoursForEmp
```

### שעות לקוח

```
getClientHours(c, mk)   = c.monthlyHours[mk] || 0
getRemainingBank(c, mk) = c.hoursBank - Σ billedHours[months before mk]
```

### הקצאות

```
getEmpAllocated(eid, mk)    = Σ matrix[mk][eid][*] (active clients only)
getClientAllocated(cid, mk) = Σ matrix[mk][*][cid]
getEmpActiveClients(eid,mk) = count of clients with allocation > 0 (max 6)
```
